# Agent Workflow Practice

Use this note to keep Bonsai Florida work fast, reviewable, and easy to resume.

## Start Every Task

- Sync from GitHub first: check the current branch, fetch, and pull the latest target branch.
- Confirm the working tree status before editing.
- Keep unrelated untracked files, such as local worktrees, out of the commit.

## Build Work

- Keep each task small and focused.
- Prefer existing project systems before creating new ones.
- For UI changes, use the fastest reliable loop: sync, edit one visible behavior, build, local page check, push.
- Avoid broad refactors when a narrow product change solves the request.

## Bonsai Florida Rules

- Do not expose the private garden address in public code.
- Public location copy should only mention West Palm Beach / Royal Palm Beach area and ZIP 33411.
- Keep booking free-first through Google Calendar Appointment Schedule unless a custom backend is explicitly requested.

## Finish Every Task

- Report what was done.
- List what changed.
- State what was tested.
- Note what remains or any blocker.
- Add a short better-practice note.
- Add the next useful thing to do, ideally a simple live-site or mobile check.
